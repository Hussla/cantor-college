## Design Rationale

### Information Architecture
- Clear separation between Computing and Design courses
- Detailed course cards with key information
- Comprehensive table for quick comparison
- Clear application process guidance

### User Experience
- Progressive disclosure of information
- Scannable course cards with bullet points
- Responsive grid layout adapting to screen size
- Clear call-to-action for applications

### Accessibility
- Semantic HTML structure with proper headings
- High contrast colors for readability
- Keyboard navigation support
- Screen reader friendly table structure

### Content Strategy
- Course descriptions focus on outcomes and benefits
- Key details (duration, entry requirements) prominently displayed
- Industry partnerships and accreditations highlighted
- Clear pathway from interest to application
